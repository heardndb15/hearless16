# Градиентные акценты для лендинга — дизайн

## Проблема

Дизайн лендинга сейчас плоский: светло-серый фон (`--bg: #F4F7FB`), белые карточки (`--bgCard: #FFFFFF`), один сплошной синий акцент (`--accent: #1565C0`). Ощущается как "просто белое".

## Решение

Добавить градиентные акценты (синий → бирюза) в ключевые точки внимания — кнопки, hero, выделенные блоки — не трогая фон обычных страниц и карточек. Это даёт больше "живости" без риска потерять читаемость на длинных текстовых страницах (блог, условия, дашборд с таблицами).

## Токены

В `landing/app/globals.css`, рядом с существующими `:root` переменными:

```css
--gradient-accent: linear-gradient(135deg, #1565C0, #22C1C3);
--gradient-soft: linear-gradient(135deg, #EAF4FF, #D9F5F0);
```

- `--gradient-accent` — насыщенный, для кнопок и текста-акцента.
- `--gradient-soft` — светлый, для фоновых панелей (hero-блоки, welcome-баннеры), контраст текста поверх сохраняется как на светлом фоне.

`--accent` (сплошной цвет) остаётся без изменений — используется в местах, где нужен сплошной цвет (текст, иконки, бордеры), а не фон.

## Точки изменения

1. **`.btn-primary`** (`globals.css`) — `background: var(--accent)` → `background: var(--gradient-accent)`. Один центральный класс, используется в 9 файлах (ai-tutor, camera-to-text, sign-language/practice, subtitles, text-to-sign, text-to-speech, `Register.tsx`, `SubtitleDemo.tsx`, `TextToSpeechSection.tsx`) — правится один раз.
2. **`.gradient-text`** (`globals.css`) — сейчас `color: var(--accent)` (плоский цвет несмотря на название класса). Меняется на настоящий градиентный текст:
   ```css
   .gradient-text {
     background: var(--gradient-accent);
     -webkit-background-clip: text;
     background-clip: text;
     color: transparent;
   }
   ```
   Уже используется в `Hero.tsx` (заголовок) — эффект подхватится автоматически, без правки JSX.
3. **`Hero.tsx`** — основная CTA-кнопка задана инлайн-стилем (`background: "var(--accent)"`, не через `.btn-primary`) → `background: "var(--gradient-accent)"`.
4. **`CTASection.tsx`** — то же самое: инлайн CTA-кнопка → `var(--gradient-accent)`.
5. **`PricingSection.tsx`** — у рекомендованного тарифа (Pro) добавляется градиентный акцент (бейдж/верхняя полоса карточки). Остальные тарифные карточки не меняются.
6. **Dashboard welcome-баннер** (`app/dashboard/page.tsx` или соответствующий layout-компонент) — фон панели переключается на `var(--gradient-soft)`.

## Что не меняется

- Фон страниц (`--bg`), обычные карточки (`--bgCard`), таблицы, длинные текстовые страницы (блог, terms, pricing-таблица целиком) — остаются светлыми/белыми, как сейчас.
- `--accent` как сплошной цвет — не трогаем, используется повсеместно для текста/иконок/бордеров.
- Тёмная тема `/sign-language/practice` (canvas-трекер, `#090d16`) — вне скоупа, не связана с этой правкой.

## Проверка

- `npx tsc --noEmit` и `npm run build` в `landing/` после правок.
- Визуально свериться со скриншотом варианта A из брейнсторминга (мягкий градиент, кнопки/hero/один выделенный блок).

## Вне скоупа

- Тёмная тема сайта целиком.
- Изменение фона обычных карточек/списков (мобильное приложение здесь не участвует — правка только `landing/`).
